<objective>
Make coverage-gap analysis asset-aware (ASSET-01, ASSET-02): ground-based runs keep claiming
every date in their window (conservative, avoids double-booking), while space-mission runs
claim nothing until their window narrows to a single concrete night. Un-narrowed space-mission
runs land in a new, distinct pending_narrowing_runs bucket (D-09) that the gap page surfaces
separately from the existing undated_runs bucket, so staff can tell "no info at all" apart from
"a real space-mission run with a range, just not scheduled tight enough yet."

Purpose: A space observatory (Hubble/JWST/Roman) has no fixed horizon, so claiming every date
in a broad window would wrongly mark those nights as covered. This closes ASSET-02's correctness
requirement using the exact Observatory.observations_type == SATELLITE_OBSTYPE check Phase 19's
D-06 calendar projection already established (ASSET-01, no new CampaignRun field).

Output: claimed_dates() 4-tuple with pending_narrowing_runs; _compute_gap() dict key; a new
gap-analysis alert block; updated + new tests.
</objective>
