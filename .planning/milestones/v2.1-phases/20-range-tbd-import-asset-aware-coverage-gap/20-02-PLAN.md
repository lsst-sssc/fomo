<objective>
Add the persistence and display foundation for the range/TBD import feature (IMPORT-01,
IMPORT-02): two new CampaignRun fields — original_obs_date_raw (the verbatim sheet text for a
row that lands as TBD) and window_needs_review (a boolean flag mirroring site_needs_review) —
plus migration 0006, and surface original_obs_date_raw as a hover tooltip on the campaign
table's TBD badge (D-08). This plan defines and displays the fields; Plan 03 (the import
command) populates them.

Purpose: IMPORT-02 requires unparseable Obs. Date rows to be flagged "needs review" and never
silently dropped — window_needs_review is that flag, and original_obs_date_raw preserves exactly
what the sheet said (e.g. "TBD pending Cycle 2", "2025-12-?") so staff can resolve it later.
D-08 makes that raw text visible without new display machinery, reusing render_site()'s tooltip
convention.

Output: two model fields, migration 0006, a TBD-badge tooltip, and tests.
</objective>
