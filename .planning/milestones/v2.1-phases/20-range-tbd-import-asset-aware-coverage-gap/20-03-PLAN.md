<objective>
Make the range/TBD import work end-to-end (IMPORT-01, IMPORT-02): extend parse_obs_window()'s
pattern-per-shape discipline to the Obs. Date column so date ranges import into the window
representation and genuinely-unparseable text imports as a flagged TBD row instead of being
skipped, and rewrite import_campaign_csv's row loop to persist and count those outcomes correctly.

Purpose: The real 3I/ATLAS sheet has rows the old importer silently skips — date ranges (space
missions with a window, not a night) and TBD free text (schedule still pending). D-13 changes
parse_obs_window()'s contract to never raise on Obs. Date, and D-06/D-07 make every Obs. Date
outcome create or update a row (never skipped), with window_needs_review flagging the ones staff
must resolve. Pitfall 2's resolved-vs-TBD natural-key branch keeps re-imports from either
duplicating TBD rows or throwing an uncaught IntegrityError.

Output: extended parser (two new regexes + rollover + never-raise 7-tuple), rewritten import
command loop, and a full set of updated + new tests.
</objective>
