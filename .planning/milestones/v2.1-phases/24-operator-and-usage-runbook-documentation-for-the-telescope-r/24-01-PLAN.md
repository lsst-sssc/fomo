<objective>
Publish a discoverable, task-oriented operator runbook for the telescope-runs-calendar
management commands and the approval-queue status-change actions — the general
"how-to-run" documentation that does not currently exist beyond design rationale
(`docs/design/*.rst`) and in-code `--help` text.

Deliver three coordinated documentation changes: (1) a Django/manage.py onboarding
subsection appended to `docs/installation.rst`; (2) a new consolidated runbook page
under `docs/runbooks/` with task-oriented "How do I...?" sections for all five
commands (plus the folded-in mark_cancelled/mark_weathered staff actions), a
quick-reference cheat-sheet, and a troubleshooting section built from real observed
failure modes; and (3) a toctree line wiring the runbook into `docs/index.rst`.

Purpose: Close the operator-knowledge gap flagged during the PR #41/#43 split review —
design docs and demo notebooks existed, but no discoverable how-to-run documentation did.
Output: Two modified `.rst` files and one new `.rst` file, all building through the
existing Sphinx pipeline (pre-commit `sphinx-build` hook) with no orphan warnings.

Scope note (docs-only, no source change): This phase makes ZERO changes to the
behavior of `telescope_runs.py`, `load_telescope_runs.py`,
`sync_lco_observation_calendar.py`, or `sync_gemini_observation_calendar.py`. Per
RESEARCH.md Pitfall 3, CLAUDE.md's "demo notebook companion" rule does NOT apply here —
no paired-notebook update is required because no listed module's behavior changes.

Open questions resolved by this plan (both were flagged in RESEARCH.md and MUST be
explicit, not left ambiguous):
