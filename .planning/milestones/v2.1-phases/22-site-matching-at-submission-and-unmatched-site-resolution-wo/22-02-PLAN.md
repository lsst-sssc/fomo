<objective>
Wire the Plan 01 live-search endpoint into the two site-entry surfaces: the public submission
form's `site_raw` field (D-09 — no create-new link for public submitters) and the approval-queue
pending row's inline site input (D-10 — replace the static ≤5-candidate `<datalist>` with the live
widget, keeping the "Create new Observatory" staff escape hatch). Both are text-only fills: picking
a suggestion populates the input; site resolution still happens at approval (D-06), so no model
change, no migration, and no DB write is triggered by anonymous typing.

Purpose: A submitter/staff member can now search by typing text *different* from the original
`site_raw` and get live substring-first suggestions (correcting a submitter's typo, finding a site
by partial name), instead of Phase 21's one-shot datalist computed against the originally-submitted
text.

Output: `CampaignRunSubmissionForm.site_raw` gains `hx-*` widget attrs + a crispy suggestions
container; `ApprovalQueueTable.render_site()`'s pending-row branch swaps its `<datalist>` for the
live HTMX widget; markup regression tests on both surfaces.
