<objective>
Close UAT gap 2, sub-issue B: a Sites Needing Review row whose site is a PLACEHOLDER
Observatory (created by `resolve_site()`'s tier-3 fallback — name `NEEDS REVIEW: <code>`,
blank timezone) offers no way to correct it. `ApprovalQueueTable.render_site()` treats "site is
set" as "genuinely resolved, only needs a projection retry" and renders plain text; and
`CampaignRunDecisionView._resolve_site()` refuses to touch any run whose `site` is already set
(the D-06 never-re-resolve guard). Both were correct for the projection-failed-retry case they
were designed for (22-REVIEWS.md finding 8c), but neither distinguishes a genuine resolution
from a placeholder. The confirmed real example: `Observatory(obscode='DCT')` has
`name='NEEDS REVIEW: DCT'` and blank timezone — a placeholder, permanently uncorrectable
through the UI.

Purpose: add a placeholder-correction escape hatch that (a) is detectable via a shared helper
+ prefix constant, (b) surfaces the live-search widget for a placeholder-site row, and (c)
lets the resolve_site action replace a placeholder site — while carefully preserving D-06's
protection against racing POSTs on a genuinely-unresolved run AND its guard against
re-resolving a genuinely-resolved site, plus the CR-01 non-reverting projection-failure path
and the D-09 never-fabricate-from-public-input invariant.

Output: `NEEDS_REVIEW_NAME_PREFIX` + `is_placeholder_observatory()` in campaign_utils.py, a
placeholder branch in `render_site()`, a placeholder-replacement path in `_resolve_site()`,
and covering tests.
</objective>
