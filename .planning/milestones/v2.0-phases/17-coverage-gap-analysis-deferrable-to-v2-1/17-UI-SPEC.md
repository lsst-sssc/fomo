## Spacing Scale

Declared values (Bootstrap 4's actual spacer utility scale — a documented exception to the generic 8-point scale below):

| Token | Value | Bootstrap utility | Usage |
|-------|-------|--------------------|-------|
| xs | 4px | `mb-1` / `p-1` | Icon gaps, inline padding |
| sm | 8px | `mb-2` / `p-2` | Compact element spacing (e.g. between a label and its helper text) |
| md | 16px | `mb-3` / `p-3` | Default element spacing; card/filter-panel padding (`card bg-light p-3`, already used in `campaignrun_table.html`) |
| lg | 24px | `mb-4` | Page-header spacing — gap between title+CTA row and the content below (used pervasively in all three existing campaign templates) |
| 2xl | 48px | `mb-5` / `p-5` | Empty-state block padding (`campaign_list.html`'s "No campaigns yet" block) |

Exceptions: **Bootstrap 4's spacer scale has no native 32px or 64px step** (it jumps `24px → 48px`). This phase must not introduce custom CSS to fill that gap — default to the nearest existing utility (`mb-4`/24px or `mb-5`/48px) exactly as the rest of this codebase already does. Do not add inline `style="margin: 32px"` or a new SCSS variable for this phase.

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Meta/caption | 14px (`<small>` / `.small`) | 400 | 1.5 |
| Body | 16px (Bootstrap default) | 400 | 1.5 |
| Section heading (h5) | 20px | 700 (`font-weight-bold`, not Bootstrap's default h5 weight-500) | 1.2 |
| Page title (h4) | 24px | 700 (`font-weight-bold`) | 1.2 |

Exactly 4 sizes (14/16/20/24), exactly 2 weights (400/700) — matches this codebase's existing convention of overriding Bootstrap's default heading weight with an explicit `.font-weight-bold` class (see `campaignrun_table.html:7`, `campaign_list.html:6`).

**Usage in this phase:**
- Page title (24px/700): `"{{ campaign.name }} — Coverage Gaps"` on the new gap-analysis page.
- Section heading (20px/700): `"No gaps found"` empty-state heading; `"Needs review: undated runs"` flag heading.
- Body (16px/400): form labels, gap-date list items, form intro paragraph.
- Meta/caption (14px/400): "Last computed at" timestamp, skipped-date caveat, D-14 disabled-button helper text.

## Copywriting Contract

| Element | Copy |
|---------|------|
| Primary CTA (table page → gap view, D-09) | **"Show Coverage Gaps"** — plain-text `btn-primary` button, no icon (matches the existing icon-free "Submit a Run"/"Filter" button convention in this app; do not add a Font Awesome icon here) |
| Secondary CTA (gap view, re-run with new params) | **"Update Results"** |
| Cancel/back link | **"Back to Campaign Table"** (`btn-outline-primary`, matches `campaignrun_submit_form.html`'s existing "Cancel" link style) |
| Empty state heading | **"No gaps found"** |
| Empty state body | "Every observable night in this window is already claimed by an approved run. Try widening the date range (up to 180 days) or choosing a different site." |
| Wait-state notice (Pitfall 1 — MUST appear before submission, not as a spinner) | "Computing observable nights can take up to a minute for a full 180-day window. Results are cached for one hour afterward." — placed as static help text directly under the "Update Results" button, visible **before** the user clicks it (see Interaction Notes below; D-09 locks a normal non-htmx GET, so there is no client-side spinner to replace with this copy). |
| "Last computed at" display (D-10) | "Last computed: {computed_at, e.g. 2026-07-04 18:32 UTC}" — 14px muted caption directly above the gap-dates list |
| "Undated, needs review" flag (D-08) | Heading: **"Needs review: undated runs"**. Body: "{{ count }} run(s) for this campaign have no observation date or start time and could not be checked against these dates." Rendered as an `alert-warning` block below the gap-dates list, never silently dropped. |
| "Observability unknown" caveat (D-03) | "{{ n }} date(s) could not be evaluated for this site and are excluded from this list." — 14px muted caption, shown only when `n > 0` (omit entirely when zero; do not show "0 dates were unknown"). |
| Out-of-scope target/site error (Pitfall 3, IDOR — 400 response) | "That target or site isn't part of this campaign. Choose one from the list below." — rendered as a form-level `alert-danger` re-showing the selection form; never a raw Django debug/400 page. |
| D-14 disabled/hidden button state | When hidden per D-14, show nothing (no dead button). When shown-but-disabled (Claude's Discretion: prefer disabled-with-explanation over fully hidden, so the user understands *why*), use: "Coverage-gap analysis needs at least one campaign target and at least one run with a resolved site." as 14px muted helper text beside the disabled button. |
| Destructive confirmation | **Not applicable.** This phase has no destructive actions — it is advisory/read-only display. Claiming a surfaced gap happens via the existing Phase 16 submission form, which is out of this phase's scope. |

## Interaction States (Phase-Specific)

These states are not covered by the generic template sections above but are load-bearing for this phase — the executor must implement all of them, and the checker must verify each explicitly.

| State | Trigger | Required Behavior |
|-------|---------|--------------------|
| Button hidden/disabled | D-14: campaign has zero `Target`s, or no `CampaignRun` has a resolved `site` | Do not render a clickable "Show Coverage Gaps" button that will fail on click. Prefer **disabled + explanatory helper text** (see Copywriting Contract) over fully hidden, so the control's absence is legible rather than silently missing. |
| Cache hit | `cache.get(key)` returns a value (D-10) | Render the cached result immediately with its original `computed_at` timestamp — do not silently update `computed_at` to "now" on a cache hit; the timestamp must reflect when the computation actually ran. |
| Cache miss (cold compute) | `cache.get(key)` is `None` | Full synchronous computation runs (worst case ~94s for a 180-day window per RESEARCH.md Pitfall 1). The wait-state notice (Copywriting Contract) must be visible on the *request* form **before** the user submits — because D-09 mandates a plain non-htmx GET, the browser's native tab/address-bar loading indicator is the only in-flight feedback; do not attempt to fake a spinner with JS this phase doesn't otherwise use. |
| Out-of-scope `target_pk`/`site_pk` submitted (Pitfall 3) | Server-side validation fails (value not in this campaign's allowed target/site sets) | Return `400 Bad Request`, re-render the selection form with the error copy above — never proceed to run the query with an unvalidated value, never redirect to a generic error page. |
| Per-date `sun_event()` `ValueError` (D-03) | A given date in range raises | Silently exclude that date from `gap_dates`/`observable_dates`; surface the aggregate count via the "observability unknown" caveat, never as a per-date error in the UI. |
| Undated `CampaignRun` present (D-08) | A run has neither `obs_date` nor derivable `ut_start` | Always show the "Needs review: undated runs" block when count > 0; never fold these rows silently into either the claimed or gap set. |

**Layout decision (Claude's Discretion, per CONTEXT.md):** The gap-analysis result renders on its **own page** (own URL, e.g. `campaigns:gap_analysis`), not as a section appended to `campaignrun_table.html`. Rationale: matches this codebase's existing pattern of one page per user-initiated action (`campaignrun_submit_form.html` is already its own page reached via a button on `campaign_list.html`), keeps the potentially ~1-minute-loading page isolated from the fast-loading table view, and avoids conditionally hiding/showing a large results block on a page that must otherwise stay fast (RESEARCH.md's anti-pattern: "never compute inline in the table view").
