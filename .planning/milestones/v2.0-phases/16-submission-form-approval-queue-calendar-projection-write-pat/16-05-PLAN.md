<objective>
Close the single minor UAT gap from 16-UAT.md Test 14: the staff approval queue forces horizontal
scrolling past 16 mostly-blank columns to reach the Approve/Reject buttons, because
`ApprovalQueueTable` inherits `CampaignRunTable`'s full 16-column spreadsheet-parity field set
verbatim and appends `actions` as the 17th (last) column.

Fix, scoped to `ApprovalQueueTable` only: trim the three columns that have no submission-form field
(and are therefore structurally always blank on a PENDING_REVIEW row) via `Meta.exclude`, and move
the `actions` column to the front via `Meta.sequence`. `CampaignRunTable` must stay untouched — it is
Phase 15's read path and must remain spreadsheet-parity per its own D-09 requirement.

Purpose: Make the pending-review queue a triage-focused view where Approve/Reject is immediately
reachable, without altering the Phase 15 read-path table.
Output: An `exclude` + `sequence` override on `ApprovalQueueTable.Meta`, plus a column-contract test.
</objective>

complete). It addresses exactly one diagnosed UAT gap (severity: minor):

- **UAT Test 14** (16-UAT.md): "For the pending review entries, is there a way to hide blank
  entries/columns so that Actions column appears with less scrolling."
- **Root cause** (`.planning/debug/approval-queue-actions-scroll.md`): `ApprovalQueueTable`
  subclasses `CampaignRunTable` with `class Meta(CampaignRunTable.Meta): pass`, inheriting all 16
  `Meta.fields`, then adds `actions` as a declared column that renders last (17th). Three of those

  not-yet-observed pending run. django-tables2 renders every declared column regardless of content,
  and the inherited `bootstrap4-responsive` template wraps the wide table in a `table-responsive`
  horizontal-scroll div. Actions being column 17 means scrolling past 16 (mostly-blank) columns.

**Scope guardrails (do NOT violate):**
- Touch `ApprovalQueueTable` ONLY. Do not edit `CampaignRunTable`, its `Meta`, or any of its
  `render_*`/column declarations — it must stay spreadsheet-parity for Phase 15's D-09.
- Do not edit `src/templates/campaigns/approval_queue.html` — the fix lives entirely in the table
  class, keeping `{% render_table %}` intact. No CSS/sticky-position change is needed because moving
  `actions` to the front already makes it reachable without scrolling (the more idiomatic
  django-tables2 fix vs. CSS sticky).
- `campaign_tables.py` is NOT one of CLAUDE.md's demo-notebook companion modules
  (only `telescope_runs.py` + the 3 management commands are), so NO notebook update is required.
