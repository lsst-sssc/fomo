<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| non-staff request → approval queue / decision endpoint | Privilege boundary — pending submissions include contact PII |
| staff POST → CampaignRun state transition | Concurrent double-approve race (TOCTOU) |
| approval action → CalendarEvent namespace | Campaign events must not collide with facility syncs |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-16-02 | Tampering | approve/reject atomic transition | high | mitigate | Single conditional `.filter(pk=pk, approval_status=PENDING_REVIEW).update(...)`; double-approve matches 0 rows (proven by `test_double_approve_is_noop`); projection gated on `updated_count == 1` |
| T-16-03 | Information Disclosure / Elevation of Privilege | `ApprovalQueueView`, `CampaignRunDecisionView` | high | mitigate | `StaffRequiredMixin` hard gate on both; anonymous/non-staff redirected to LOGIN_URL (proven by `TestStaffGating`) — this page must NOT follow Phase 15's soft-filter pattern (Pitfall 7) |
| T-16-05 | Tampering | approve/reject POST forms | medium | mitigate | `CsrfViewMiddleware` active + `{% csrf_token %}` in each per-row form |
| T-16-06 | Tampering | GET-triggered state change | medium | mitigate | `http_method_names = ['post']` on the decision view — a crawler prefetch/`<a href>` cannot approve |
| T-16-09 | Tampering | CalendarEvent namespace collision | medium | mitigate | Key `CAMPAIGN:{pk}` via `insert_or_create_calendar_event` — distinct namespace from LCO/Gemini/classical `url` keys (Pitfall 3) |
| T-16-SC | Tampering (supply chain) | package installs | low | accept | No new packages this phase |
</threat_model>
