<objective>
Build the bootstrap CSV import: a `campaign_utils.py` helper module (site resolution, best-effort
UT-time parsing, Observation-Status translation, no-churn create-or-update) and an
`import_campaign_csv` management command that ingests the real 3I/ATLAS coordination-sheet CSV
row-by-row with skip-and-log error handling and a created/updated/unchanged/skipped/site_needs_review
summary. Cover it with Django tests that mock the MPC API.

Purpose: This is CAMP-04 — validating the CampaignRun schema against the real, messy sheet is the
whole reason this phase is ordered first. The command must tolerate the sheet's real messiness
(colon typos, approximate times, blank/oversized site codes, coarse status strings) without dropping
rows it shouldn't.
Output: `campaign_utils.py`, `import_campaign_csv.py`, `test_import_campaign_csv.py`.

Operator follow-up (not part of this plan's automated verification): the actual one-off live import
(CAMP-04's real run) requires the operator to export the real 3I/ATLAS sheet to CSV and run
`import_campaign_csv --campaign "3I/ATLAS" <exported.csv>`. That file is never committed and is never
a test/notebook dependency.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| operator CSV file → parser | Untrusted, messy free-text cells cross into DB writes via the import command |
| import command → MPC Obscodes API | One outbound HTTPS GET per unresolved, valid-length Site Code |
| CLI filepath → filesystem | Operator supplies an arbitrary path to read (CLI trust boundary, not web) |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-14-01 | Tampering | CSV field parsing (V5 input validation) | medium | mitigate | Length-check Site Code against `Observatory.obscode.max_length` before any DB write (no truncation/fabrication); targeted regex for UT time (no permissive parser that "succeeds" on garbage); status translation with a safe REQUESTED default; skip-and-log only on natural-key failure so a bad cell never aborts the run |
| T-14-03 | Information Disclosure | `import_campaign_csv <filepath>` reading arbitrary paths | low | accept | Same accepted trust boundary as the existing `load_telescope_runs <filepath>` CLI; operator-run tool, not a web upload; `open()` wrapped in OSError->CommandError |
| T-14-04 | Tampering / Information Disclosure | SSRF via Site Code into MPCObscodeFetcher | low | accept | `requests.get` targets a hardcoded MPC URL with obscode passed as a JSON body param (not URL-interpolated) — no attacker-controlled URL construction; input is operator-supplied, not web-form data |
| T-14-06 | Tampering | CSV/spreadsheet formula injection in free-text cells (e.g. `=HYPERLINK(...)`) | low | accept | This phase only imports CSV, never re-exports to a spreadsheet — no injection sink introduced here; flag for any future CSV-export phase |
| T-14-SC | Tampering | pip installs | low | accept | No new packages — stdlib (`csv`, `re`, `datetime`) + already-installed only (RESEARCH Package Legitimacy Audit) |
</threat_model>
