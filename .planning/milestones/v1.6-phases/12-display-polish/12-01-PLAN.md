<objective>
Two surgical calendar-UI fixes carried over as deferred debt from v1.3ŌĆōv1.5:

1. **DISPLAY-08** ŌĆö Compute all-day calendar event text color from the WCAG 2.1 relative
   luminance of its proposal palette background (white or black, whichever achieves >= 4.5:1
   contrast), replacing the hardcoded `color: #fff !important` CSS rules. Formula-driven so any
   future palette change is automatically correct.

2. **DISPLAY-09** ŌĆö Eliminate two N+1 query patterns in the calendar event render loops
   (`telescope_label_meta.is_verified` reverse-OneToOne accessor and `active_todos.count`
   filtered aggregate) via a FOMO-local wrapper view that shadows `tom_calendar.render_calendar`
   for the `/calendar/` URL only, adding `prefetch_related` + a `Count` annotation.

Purpose: Clear the last display-polish debt items before Stage 4, with accessibility and
query-count both verified by the test suite.

Output: New `text_color_for_bg` template tag, new `fomo_render_calendar` view + `calendar_urls.py`,
URL ordering change, template edits, and unit + integration tests.
</objective>

| T-12-03 | Denial of Service | calendar event render loop (prior N+1) | mitigate | This phase removes the O(N) query growth: `prefetch_related('telescope_label_meta')` + `Count` annotation make the page query count O(1) in event count, shrinking rather than expanding the DoS surface of large months. |
| T-12-04 | Information Disclosure | `fomo_render_calendar` query params (`month`/`year`/`utc_offset`) | accept | Parsing/bounds are inherited verbatim from upstream `render_calendar` (e.g. `month = max(1, min(12, month))`); this phase changes only the queryset chain, introducing no new untrusted-input path. |
| T-12-SC | Tampering | npm/pip/cargo installs | accept | No packages are installed in this phase (zero new dependencies ŌĆö all functionality from Django ORM and Python stdlib per 12-RESEARCH ┬¦Standard Stack). No legitimacy checkpoint required. |
