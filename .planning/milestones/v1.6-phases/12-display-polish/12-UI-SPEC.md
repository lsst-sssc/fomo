The following values are established and preserved. Phase 12 changes only the computed
text color on all-day event text; it does not change any size or weight.

**DISPLAY-08 text color change (new):**

- Before: `.cal-event-all-day` and `.cal-event-all-day a` both carry `color: #fff !important` in the `<style>` block; text is always white.
- After: Remove both `color: #fff !important` declarations. Text color is computed per event via `{% text_color_for_bg bg_color as text_color %}` and applied as `color: {{ text_color }};` in the inline `style` attribute on each all-day event `<div class="cal-event-all-day">`.
- For all 9 current palette entries (8 `PROPOSAL_PALETTE` + `NEUTRAL_SLOT_COLOR`), the computation returns `'#fff'` — visually identical to the current state, but formula-driven.
- **Scope exclusion:** Timed events (`.cal-event-timed`) have a transparent background. The palette color is applied only to the `▌` bullet (`style="color: {{ bg_color }};"`) not the text. No text-color change is applied to timed events.

## Color

### Bootstrap 4 base palette (existing, unchanged)

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#ffffff` | Page background; calendar day cells (current month) |
| Secondary (30%) | `#f8f9fa` (`var(--light)`) | Out-of-month day cells; header row background |
| Accent (10%) | `#007bff` (`var(--primary)`) | Today date circle; navigation links only |
| Destructive | `#dc3545` (`var(--danger)`) | No direct calendar use (terminal event states handled via box-shadow, not background) |

Accent reserved for: today date circle background (`.cal-day.today .day-num`), and hyperlink text color. Never used as an event background or legend color.

### Calendar proposal palette (existing, unchanged)

The 8 `PROPOSAL_PALETTE` hex values locked by Phase 9 (09-UI-SPEC.md). Colorblind-vetted (deuteranopia + protanopia). All 8 are dark colors; white text achieves WCAG AA against each.

| Index | Hex | Relative luminance (approx.) | Computed text color |
|-------|-----|-------------------------------|---------------------|
| 0 | `#005f9e` | 0.045 | `#fff` |
| 1 | `#a34000` | 0.077 | `#fff` |
| 2 | `#5b2080` | 0.022 | `#fff` |
| 3 | `#006b4e` | 0.050 | `#fff` |
| 4 | `#9e1c1c` | 0.043 | `#fff` |
| 5 | `#006b6b` | 0.052 | `#fff` |
| 6 | `#6b2060` | 0.023 | `#fff` |
| 7 | `#7a4500` | 0.048 | `#fff` |
| — | `#5a6268` (NEUTRAL_SLOT_COLOR) | 0.125 | `#fff` |

All 9 entries have relative luminance ≤ 0.183. White contrast ratio ≥ 4.5:1 confirmed for each. Source: RESEARCH.md §Pattern 1.

### WCAG text color computation contract (new — DISPLAY-08)

| Property | Specification |
|----------|---------------|
| Tag name | `text_color_for_bg` |
| Module | `solsys_code/templatetags/calendar_display_extras.py` |
| Decorator | `@register.simple_tag` |
| Signature | `text_color_for_bg(hex_color: str) -> str` |
| Input | `'#rrggbb'` hex color from `PROPOSAL_PALETTE` or `NEUTRAL_SLOT_COLOR` |
| Output | `'#fff'` if white achieves ≥ 4.5:1 contrast; `'#000'` otherwise |
| Formula | WCAG 2.1 relative luminance: `linearize(c) = c/255 if c/255 ≤ 0.04045 else ((c/255 + 0.055)/1.055)^2.4`; `L = 0.2126·R + 0.7152·G + 0.0722·B`; white passes when `1.05 / (L + 0.05) ≥ 4.5` |
| Template usage | `{% text_color_for_bg bg_color as text_color %}` (captured into variable) |
| Template placement | After `{% proposal_color event.proposal as bg_color %}` and `{% status_border_css event.title as status_border %}`, before the `{% if event.telescope_label_meta.is_verified == False %}` branch — placed once, in scope for both div branches |
| Applied to | All-day event `<div class="cal-event-all-day">` inline style only |
| Not applied to | `.cal-event-timed` divs (transparent background; no text-background contrast issue) |
| Not applied to | `.cal-event-all-day a` separately — the `<a>` inherits from the parent div without `!important` interference once the CSS rules are cleaned |

### CSS removal contract (DISPLAY-08)

Two `!important` color declarations must be removed from the `<style>` block in `calendar.html`:

| Location | Current declaration | Action |
|----------|---------------------|--------|
| `.cal-event-all-day` rule | `color: #fff !important;` | Remove this property declaration. The remainder of the rule is unchanged. |
| `.cal-event-all-day a` rule | `color: #fff !important;` | Remove this property declaration. The `text-decoration: none;` and `a:hover { text-decoration: underline; }` rules are unchanged. |

After removal, the inline `color: {{ text_color }};` on each event div controls the text color without `!important` fights.
