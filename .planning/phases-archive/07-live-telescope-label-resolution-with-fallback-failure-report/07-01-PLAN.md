<objective>
Build the verified site/telescope mapping dict and the new live API-resolution
helpers in `sync_lco_observation_calendar.py`, replacing the current
flat-`parameters['site']` assumption with a single, timeout-bounded LCO
Observation Portal API call that resolves a placed record's real
site/enclosure/telescope — and that fails safe (returns None, never raises,
never leaks the response body or API key) per D-10/D-11.

Purpose: This is the pure-logic core of Phase 7 (TELESCOPE-01, TELESCOPE-02,
SYNC-08, SYNC-09). It produces the helpers that Plan 02 wires into the
per-record loop. No `Command.handle()` / `_build_event_fields` wiring happens
here — that is Plan 02 (Wave 2), which depends on these helpers existing.

Output: A migrated 8-site `SITE_TELESCOPE_MAP`, `_resolve_placement_block`,
`_aperture_class_from_telescope_code`, the new 2-arg `_derive_telescope`, and
their unit tests (verified-dict coverage, aperture-class parsing,
single-attempt-no-retry, no-credential-leak).
</objective>
