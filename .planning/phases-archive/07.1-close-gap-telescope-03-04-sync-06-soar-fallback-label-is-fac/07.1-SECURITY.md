## Trust Boundaries

| Boundary | Description | Data Crossing |
|----------|-------------|---------------|
| LCO/SOAR Observation Portal API → command | Already-existing boundary, not touched by this phase. The fix executes purely on the fallback path, reached only after the API call has already failed/timed out/returned an unmapped code. ## Threat Register

| Threat ID | Category | Component | Disposition | Mitigation | Status |
|-----------|----------|-----------|-------------|------------|--------|
| T-07.1-01 | Tampering | `record.facility` value driving the SOAR branch | accept | `facility` is set by TOM Toolkit's facility dispatch (Phase 5), not by free-form user input; gated upstream by `facilities.get(record.facility)` returning non-None, itself fed only by a case-sensitive `filter(facility__in=['LCO', 'SOAR'])`. A tampered value at worst yields the LCO fallthrough (raw-string label) — the same non-fatal degrade that exists today. No escalation, no new exposure. | closed |
| T-07.1-02 | Information Disclosure | the `4m0`/`[UNVERIFIED]` fallback title + description text | mitigate | The fallback description string (`sync_lco_observation_calendar.py:457`, `'Telescope label unverified: live API lookup failed or returned an unmapped code.'`) is the existing generic, never-exception-derived message — verified untouched by this phase's diff (`git diff 0b7aea3..ed9e1cc`). No raw response body, error detail, or credential is added to the label/title/description. SYNC-09 no-leak behavior from Phase 7 is preserved unchanged. | closed |
| T-07.1-03 | Denial of Service | `_coarse_telescope_label` on the fallback path | accept | Function is O(1) string slicing/comparison (one new `==` early-return guard) with no loops, no recursion, no I/O; cannot raise and cannot block. Confirmed by code review (07.1-REVIEW.md) and the full passing test suite. | closed |
| T-07.1-SC | Tampering | npm/pip/cargo installs | accept | No package installs in this phase — no new dependencies (`requests`/`jupyter`/`django` already present); confirmed no `pyproject.toml`/lockfile changes in the phase diff. No package-legitimacy checkpoint required. | closed |
