<objective>
Replace `sync_lco_observation_calendar.py`'s single flat-key instrument read
(`instrument = record.parameters['instrument_type']`, line 143) with extraction
logic that scans the real `c_1..c_5`-prefixed multi-configuration shape, correctly
distinguishing a science config from SOAR calibration configs (ARC/LAMP_FLAT) and
detecting LCO MUSCAT's per-channel exposure shape. Add a dedicated counter for total
extraction failures, distinct from the existing `'skipped'` counter, so malformed
records are visible separately in the run summary. Extend the test fixture helper and
add tests for the SOAR multi-config, MUSCAT per-channel, and malformed-record cases.

Purpose: Fixes the v1.2 real-data bug where the command read a flat
`parameters['instrument_type']` key that does not exist on real LCO/SOAR records,
producing wrong or missing instrument labels on calendar events (EXTRACT-01, EXTRACT-02).

Output: New private helper(s) in the command module implementing D-01..D-06, a new
per-facility extraction-failure counter wired into all three counter locations, an
additive `extra_params` parameter on the test `_parameters()` fixture, and three new
tests covering the SOAR/MUSCAT/malformed shapes.
</objective>
