<objective>
Create the `solsys_code/templatetags/` package and its `calendar_display_extras.py` module — the pure-Python core of Phase 9's read-side visual encoding — plus its Wave 0 unit tests. This module exposes three Django `simple_tag`s consumed by `calendar.html` in Plan 02:

- `proposal_color(proposal)` — deterministic, colorblind-vetted, curated-palette color for a proposal code (DISPLAY-04).
- `status_border_css(title)` — title-prefix → `box-shadow` CSS string distinguishing queued / placed / terminal-failure (DISPLAY-06; mechanism D-08 resolved to box-shadow in 09-UI-SPEC.md).
- `visible_proposals(weeks)` — current-month legend data, grouped by color with collision handling (DISPLAY-07 / D-02 / D-04).

Purpose: Isolate all hashing/prefix/grouping logic in one tested Python module rather than letting `calendar.html`'s inline `{% if %}` chain grow (RESEARCH Anti-Pattern 3). This plan ships no template change — it is the dependency Plan 02 builds on.

Output: A new template-tag package, the three tags, and a passing unit-test suite.
</objective>
