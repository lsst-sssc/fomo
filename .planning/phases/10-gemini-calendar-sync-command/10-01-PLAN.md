<objective>
Implement the `sync_gemini_observation_calendar` management command and its full Django test
suite. The command reads every `ObservationRecord(facility='GEM')`, derives `CalendarEvent`
fields (telescope, instrument, proposal, title, window) from the record's `parameters` JSON plus
`settings.FACILITIES['GEM']['programs']`, and creates/updates events idempotently. No live Gemini
API calls, no sidecar model. This plan covers all 10 GEM-* requirements.

Purpose: Deliver the core Stage-4 Gemini sync capability with a test suite that pins down every
window-derivation branch, the no-churn contract, and the credential-safety guarantee.
Output: A new management command file and a new test file, both ruff-clean, all tests passing.
</objective>
