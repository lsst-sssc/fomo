<objective>
Run the live, human-in-the-loop ESO Phase 2 API investigation for Paranal (VLT)
production and capture real, redacted evidence into `13-DECISION.md`'s Findings
sections. This resolves the three empirical questions the decision (Plan 02)
depends on: are credentials obtainable/usable (ESO-01), what does a real OB
status/execution response actually look like (ESO-02), and can a headless Django
management command source credentials at all given ESO's session-bound encrypted
auth (ESO-03).

This is investigation-only. No application code and no management command are
built. The exploration happens through a throwaway scratch probe script (NOT a
committed deliverable, per D-09) that the operator runs against their real
production credentials. Only `13-DECISION.md` is committed from this plan.

Purpose: Ground Plan 02's Bridge/Bypass/Not-Yet-Feasible recommendation in real
API evidence rather than documentation guesswork — the same discipline that
caught the v1.2/v1.3 fixture-shape bugs.
Output: `13-DECISION.md` with populated Findings for ESO-01, ESO-02, ESO-03; a
throwaway `eso_p2_probe.py` (uncommitted).
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| operator's ESO credentials -> scratch probe script / shell env | Real production P2 username+password enter the local process only; must never be hard-coded, printed, or committed. |
| live p2api response -> committed `13-DECISION.md` | Untrusted-for-disclosure content (may embed usernames, sensitive program IDs) crosses into a version-controlled file; must be redacted first (D-04). |
| p2api / ESOAPI exception objects -> stdout/logs | `p2api.p2api.P2Error` messages can embed credential/connection detail; must be scrubbed, never interpolated verbatim. |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-13-01 | Information Disclosure | Verbatim API response pasted into `13-DECISION.md` | high | mitigate | D-04 redaction: replace usernames/passwords/sensitive progIds with `<REDACTED>` before writing; Task 3 human-check + grep for the `redacted per D-04` note gate the commit. |
| T-13-02 | Information Disclosure | Credentials in env/`local_settings.py` and in the scratch probe | high | mitigate | D-03: credentials only via env var or already-git-ignored `src/fomo/local_settings.py`; probe never hard-codes or prints them; `eso_p2_probe.py` added to `.git/info/exclude` so it is never staged. |
| T-13-03 | Information Disclosure | `P2Error`/connection exception text echoed to stdout/logs | high | mitigate | Probe prints only `type(err).__name__` + a fixed static note, never the raw exception object (Pitfalls security table precedent). |
| T-13-04 | Tampering | Accidental write against real production P2 API | high | mitigate | D-08 read-only guardrail: probe contains zero save/submit/create calls; Task 1 grep-gate asserts their absence. |
| T-13-SC | Tampering | Package installs (npm/pip/cargo) | low | accept | No new packages are installed this phase — `tom-eso==0.2.4`/`p2api==1.0.10` are already present and audited by prior research; nothing to install. |
