Wire the public submission path: a `CampaignRunSubmissionView` (`FormView`) that validates the
Plan 01 form, silently drops honeypot-tripped submissions, creates a `PENDING_REVIEW`
`CampaignRun`, notifies staff by email (no PII), and a generic thanks page shown identically for
genuine and bot submissions.

notification). The row lands as `pending_review` (model default) and is thus invisible on public
views once Plan 04 adds the D-09 filter.
Output: submission view + two URLs + two templates + `test_campaign_submission.py`.
</objective>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| anonymous browser → `CampaignRunSubmissionView` | Untrusted public POST creates a DB row |
| FOMO app → staff mailboxes/mail logs | Notification email crosses outside FOMO's DB-level PII gate |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-16-01 | Denial of Service / Tampering | `form_valid` honeypot short-circuit | medium | mitigate | `alt_contact_info` non-empty → return the same thanks redirect with no create/email/error signal (Pattern 3) |
| T-16-04 | Information Disclosure | `_notify_staff` email body/subject | high | mitigate | D-04 bare ping + approval-queue link only; no contact/telescope/campaign details; test asserts no PII in outbox |
| T-16-05 | Tampering | submission POST form | medium | mitigate | `CsrfViewMiddleware` active + crispy `{% crispy %}` emits `{% csrf_token %}` |
| T-16-08 | Denial of Service | `send_mail` failure breaking the request | low | mitigate | `fail_silently=True` so a mail outage never 500s the genuine submission (Pitfall 6) |
| T-16-SC | Tampering (supply chain) | package installs | low | accept | No new packages this phase |
</threat_model>
