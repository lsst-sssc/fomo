<objective>
Represent the real Gemini Fast-Turnaround program GS-2026A-FT-115 as an informational
`CampaignRun` and prove the whole storm-cancellation mechanism handles it with no special-casing
(D-06, D-07). This plan adds NO production code — every capability already exists (Plan 02's
`_set_run_status()` action, the existing `resolve_site()` Tier-2 MPC resolver, the range-window
skip-by-design projection). It delivers the reproducible test encoding of the Gemini scenario plus
an operator runbook for creating the live row.

Purpose: When the storm also wipes FT-115 (a 6.50-hour GMOS-S allocation, window 13-16 July), staff
mark it the same way as the Magellan runs. Because its window is a 4-day range, it never had a
projected calendar event — so marking it weathered must set run_status without crashing or
fabricating an event. This plan proves exactly that against the real seed values.

Output: A verified `resolve_site('I11')` test (Gemini South, ground-based, real timezone) via a new
`BULK_MPC_FIXTURE` entry, and an end-to-end Gemini scenario test (create → approve → mark weathered
→ no event fabricated). The live production row is operator data entry (see user_setup).
</objective>
