<objective>
Wire the already-parsed classical-schedule `cancelled` status word into a visible `[CANCELLED]`
title prefix on the `CalendarEvent`, matching the LCO sync's existing title-prefix idiom (D-01,
D-02). Today `load_telescope_runs.py` embeds `Status: cancelled` in the event *description* only
and never touches the *title*, so a weathered-out classical run is visually indistinguishable
from a live one on the calendar.

Purpose: A staff member marks a classical Magellan/NTT/FTS run cancelled by editing the source
schedule file (adding the `(cancelled)` parenthetical the parser already recognizes) and
re-running `load_telescope_runs` — the existing idempotent create-or-update path — after which the
calendar tile shows `[CANCELLED] {telescope} {instrument}` and gains the terminal box-shadow ring
that `[CANCELLED]` already triggers.

Output: A small status→prefix constant dict and a fresh-every-run title computation in
`load_telescope_runs.py`; new Django tests; a regenerated demo notebook cell demonstrating the
prefix. No new command, no new UI, no new model field, no templatetag change.
</objective>
