<objective>
Add an optional `--campaign` flag to the `load_telescope_runs` management command that
resolves a `tom_targets.TargetList` by name and associates it (as `CalendarEvent.target_list`)
with every CalendarEvent created or updated from the run file in one invocation. When the flag
is omitted, behavior is completely unchanged from today (target_list stays None on every event).

Purpose: bring `load_telescope_runs` in line with `backfill_lco_observation_records` and the
just-fixed `sync_lco_observation_calendar` so classical-run schedule files can be tagged to a
campaign, without introducing a per-line grammar token (classical schedule files are organized
one file per PI/project, so a single uniform flag matches real usage).

Output: updated command, extended tests, and the mandated companion demo notebook.
</objective>
