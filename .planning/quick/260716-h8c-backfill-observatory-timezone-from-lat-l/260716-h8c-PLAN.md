<objective>
Backfill `Observatory.timezone` from the resolved geodetic coordinates when a new
Observatory is created via `MPCObscodeFetcher.to_observatory()` (the Tier-2 MPC site-code
lookup path), using the `timezonefinder` library (offline IANA zone lookup from lat/lon).

Purpose: Tier-2 MPC-resolved Observatory records always get a blank `timezone` because the
MPC Observatory Codes API doesn't supply one, which makes `_project_calendar_event()`'s
calendar projection raise `ValueError` and forces a human to manually set the timezone in
the admin before a "Sites Needing Review" retry can succeed. Computing a sensible default at
creation time closes that gap so most real observatories work out of the box, while
`timezone` stays a plain editable field a human can still correct.

Output: `timezonefinder` added as a dependency; `to_observatory()` populates `timezone`
defensively (only when blank, never fabricating for unresolvable coordinates); three
regression tests covering the correct-backfill, no-clobber, and unresolvable-leaves-blank
paths.
</objective>

Key facts for the executor:
- `Observatory.timezone` is `CharField(max_length=64, blank=True, default='')` — an
  ordinary editable field. Leaving it `''` is a valid, supported state.
- In `to_observatory()`, `obs.from_parallax_constants(...)` (currently ~line 105) resolves
  `obs.lat` and `obs.lon` from the MPC parallax constants BEFORE the record is saved — this
  is the natural backfill point because both coordinates are already computed there.
- The MPC `obs_data` dict has no `timezone` key in live data (see the fixtures in
  `test_utils.py`). Read it defensively anyway so a future/injected value is respected.
- Satellite-type MPC records (250/274/289) raise a TypeError earlier at
  `float(self.obs_data['longitude'])` and never reach the backfill point — no special
  handling needed, but keep a cheap `lat is not None and lon is not None` guard.
- `timezonefinder.TimezoneFinder().timezone_at(lat=..., lng=...)` returns an IANA zone name
  (e.g. `'Australia/Sydney'`) or `None` for coordinates outside any timezone polygon.
- ruff: single quotes, 120-col, Google-style docstrings; run `ruff check .` and
