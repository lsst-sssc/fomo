# Quick Task 260719-d18: Fix calendar data-url missing utc_offset Summary

**Added `&utc_offset={{ utc_offset }}` to the `#calendar-partial` `data-url` in `calendar.html`, matching the existing Prev/Next/Today hx-get pattern, so the vendored `tom_calendar` `calRefresh` handler's reload no longer silently resets the calendar to UTC+0 after an event/todo create/update/delete.**

## Accomplishments
- Fixed the `#calendar-partial` `data-url` attribute to carry `month`, `year`, AND `utc_offset` — the same three params the Prev/Next/Today buttons already send — closing the gap where a `calRefresh`-triggered reload (fired by the vendored `tom_calendar` package after any event/todo create/update/delete) always re-requested the calendar at UTC+0.
- Added a regression test (`test_calendar_partial_data_url_carries_utc_offset`) using a non-zero offset (5) to prove the user's actual selection threads through the rendered `data-url`, not just that a literal `0` happens to appear.
