<objective>
De-emphasize `[QUEUED]` calendar events so a tentative (unscheduled) LCO observation never renders louder than a confirmed/placed one.

Purpose: `tom_calendar.models.CalendarEvent.color` is `BOOTSTRAP_COLORS[self.pk % len(...)]` — purely a function of primary key, with zero relationship to status. A `[QUEUED]` event can coincidentally get a loud color (e.g. purple) while a confirmed event gets a quiet one, inverting visual priority. Because `tom_calendar` ships bundled inside the `tomtoolkit` wheel and must not be edited in place, fix this with a LOCAL Django template override.

Output: A new file `src/templates/tom_calendar/partials/calendar.html` — a near-verbatim copy of the installed upstream template with only the all-day event background-color style changed to de-emphasize `[QUEUED]` titles.

Scope guard: ONLY the `[QUEUED]` case. Do NOT build telescope/proposal-keyed coloring or status-driven treatment for `[EXPIRED]`/`[CANCELLED]`/`[FAILED]` — that is deferred to backlog todo `.planning/todos/pending/2026-06-18-status-aware-calendar-event-coloring-telescope-proposal-keye.md`.
</objective>
